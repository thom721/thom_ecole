Pour gérer efficacement l'activation et la connexion des boutons en fonction des rôles multiples d'un utilisateur dans une application PyQt5, vous pouvez adapter votre code comme suit :


---

1. Définir les rôles et les boutons associés

Commencez par définir les rôles et les boutons qui leur sont associés :

roles_boutons = {
    'admin': [
        'btn_left_home', 'btn_left_admin', 'btn_left_etudiant', 'btn_left_paiement',
        'btn_left_prof', 'btn_left_cours', 'btn_left_promus', 'btn_left_rapport',
        'btn_left_vente', 'btn_left_profile', 'btn_settings', 'btn_actualiser'
    ],
    'professeur': ['btn_left_notes'],
    'caissier': ['btn_left_paiement', 'btn_left_vente'],
    'responsable_pedagogique': ['btn_left_paiement', 'btn_left_vente', 'btn_left_etudiant']
}


---

2. Identifier les rôles de l'utilisateur

Ensuite, obtenez les rôles de l'utilisateur actuel. Si un utilisateur peut avoir plusieurs rôles, vous pouvez les stocker dans une liste :

roles_utilisateur = ['professeur', 'caissier']  # Exemple : l'utilisateur a les rôles 'professeur' et 'caissier'


---

3. Activer ou désactiver les boutons en fonction des rôles

Parcourez tous les boutons et activez-les ou désactivez-les en fonction des rôles de l'utilisateur :

# Liste de tous les boutons
btn_left = [
    'btn_left_home', 'btn_left_admin', 'btn_left_etudiant', 'btn_left_paiement',
    'btn_left_prof', 'btn_left_cours', 'btn_left_promus', 'btn_left_rapport',
    'btn_left_vente', 'btn_left_profile', 'btn_settings', 'btn_actualiser'
]

# Dictionnaire des callbacks de boutons
button_callbacks = {
    'btn_left_home': self.dash_page,
    'btn_left_admin': self.admin_page,
    'btn_left_etudiant': self.etudiant_page,
    'btn_left_paiement': self.paiement_page,
    'btn_left_prof': self.professeur_page,
    'btn_left_cours': self.cours_page,
    'btn_left_notes': self.notes_page,
    'btn_settings': self.settings_page,
    'btn_actualiser': self.actualiser_page,
    'btn_left_promus': self.promus_page,
    'btn_left_rapport': self.rapport_page,
    'btn_left_vente': self.vente_page,
    'btn_left_profile': self.profile,
}

# Activer ou désactiver les boutons en fonction des rôles
for btn_name in btn_left:
    btn = getattr(self.ui, btn_name)
    if any(role in roles_utilisateur for role in roles_boutons if btn_name in roles_boutons[role]):
        btn.setEnabled(True)
        btn.clicked.connect(button_callbacks[btn_name])
    else:
        btn.setEnabled(False)

# Rediriger vers la page par défaut
if 'admin' in roles_utilisateur:
    self.ui.main_with_shadow.setCurrentIndex(4)
    self.ui.stackedWidget.setCurrentIndex(0)
else:
    self.ui.stackedWidget.setCurrentIndex(7)
    self.ui.btn_left_notes.clicked.connect(self.notes_page)


---

4. Explication du code

roles_boutons : Un dictionnaire associant chaque rôle à une liste de boutons activés pour ce rôle.

roles_utilisateur : La liste des rôles de l'utilisateur actuel.

btn_left : La liste de tous les boutons disponibles.

button_callbacks : Un dictionnaire associant chaque bouton à sa méthode de callback correspondante.

Boucle for : Parcourt chaque bouton et l'active ou le désactive en fonction des rôles de l'utilisateur.



---

5. Remarques supplémentaires

Gestion des rôles multiples : Si un utilisateur peut avoir plusieurs rôles, le code vérifie si l'un de ses rôles lui permet d'accéder à un bouton donné.

Utilisation de blockSignals(True) : Si vous souhaitez empêcher temporairement la connexion d'un signal à un bouton, vous pouvez utiliser btn.blockSignals(True) pour désactiver les signaux du bouton. Cela peut être utile si vous devez effectuer des opérations sans déclencher d'événements liés aux boutons.


N'hésitez pas à ajuster ce code en fonction de la structure exacte de votre application et des besoins spécifiques de votre projet.
