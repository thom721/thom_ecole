```python
import subprocess

# Paramètres


def certificat_existe(self ,subject_name="CN=app.exe"):
    """
    Vérifie si le certificat existe déjà dans Cert:\LocalMachine\Root
    """
    cmd = [
        "powershell",
        "-Command",
        f"Get-ChildItem -Path Cert:\\LocalMachine\\Root | Where-Object {{ $_.Subject -eq '{subject_name}' }}"
    ]
    try:
        result = subprocess.run(cmd, capture_output=True, text=True, check=True)
        return bool(result.stdout.strip())  # True si quelque chose est trouvé
    except subprocess.CalledProcessError as e:
        print("Erreur lors de la vérification du certificat :", e.stderr)
        return False

def installer_certificat(self):
    """
    Installe le certificat PFX dans Cert:\LocalMachine\Root
    """
    PFX_PATH = r"C:\Program Files\gestion ecole\cert\codesign-test.pfx"
     PFX_PASSWORD = "@#Janvier21"
    cmd = [
        "powershell",
        "-Command",
        (
            f"$Password = ConvertTo-SecureString '{PFX_PASSWORD}' -AsPlainText -Force; "
            f"Import-PfxCertificate -FilePath '{PFX_PATH}' "
            f"-CertStoreLocation 'Cert:\\LocalMachine\\Root' "
            f"-Password $Password | Out-Null"
        )
    ]
    try:
        subprocess.run(cmd, capture_output=True, text=True, check=True)
        print("[OK] Certificat installé avec succès ✅")
    except subprocess.CalledProcessError as e:
        print("[ERREUR] Échec de l'installation du certificat ❌")
        print(e.stderr)

if __name__ == "__main__":
    print("=== Vérification du certificat ===")
    if certificat_existe():

    else:
        print("[INFO] Certificat absent → Installation en cours...")
        installer_certificat()
```

3. Le script s’exécute silencieusement → si le certificat est déjà installé, rien n’est refait.

⚠️ **Important :** il faut lancer ton script en **administrateur** pour que `LocalMachine\Root` accepte l’installation.

## 1️⃣ Faire ça en **CMD** (sans PowerShell)

En **CMD pur**, tu ne peux pas générer de certificat comme avec `New-SelfSignedCertificate`.
Mais tu peux utiliser **`makecert.exe`** (ancien, retiré de Windows SDK) ou mieux **`certutil`** (inclus dans Windows).

👉 Exemple avec `certutil` :

```bat
:: Crée un certificat auto-signé dans le magasin CurrentUser\My
certutil -scinfo

:: Génération
certutil -user -selfsign "CN=app.exe" ^
  -r -pe -n "app Code Signing" ^
  -eku 1.3.6.1.5.5.7.3.3 ^
  -sv app.pvk app.cer

:: Export en PFX
pvk2pfx -pvk app.pvk -spc app.cer -pfx app.pfx -po @#Janvier21
```

⚠️ Le hic :

* `pvk2pfx` vient aussi du Windows SDK, pas présent par défaut.
* Donc **PowerShell reste plus simple** pour générer/ exporter un PFX.

## 2️⃣ Faut-il installer le certificat chez les autres utilisateurs ?

Oui ✅

* Ton exécutable **signé avec un certificat auto-signé** ne sera pas reconnu comme "éditeur de confiance" par Windows.
* Chaque utilisateur (chaque PC, pas seulement chaque session) doit **installer ton certificat racine** dans le **Magasin "Autorités de certification racines de confiance"** pour éviter l’avertissement *"Éditeur inconnu"*.

👉 Commande d’installation pour tous les utilisateurs (Admin requis) :

```bat
certutil -addstore -f "Root" C:\cert\codesign-test.pfx
```

Ou via PowerShell :

```powershell
Import-PfxCertificate -FilePath "C:\cert\codesign-test.pfx" `
  -CertStoreLocation "Cert:\LocalMachine\Root" `
  -Password (ConvertTo-SecureString "@#Janvier21" -Force -AsPlainText)
```

  * Chaque **machine cliente** doit installer ton certificat (dans *Autorités racines de confiance*).
* Si tu veux éviter ça, il faut acheter un **vrai certificat Code Signing** auprès d’une autorité (DigiCert, Sectigo, GlobalSign…).

        )

        # 3. Vérification
